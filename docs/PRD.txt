# Preflight — Product Requirements Document
**Version:** 0.1.0-alpha  
**Status:** Active  
**Stack:** React + Vite + TypeScript + Tailwind + shadcn/ui + Dexie.js + Zustand + Supabase (optional)

---

## 1. Product Overview

Preflight is an open-source Project Operating System for vibe coders — builders who use AI coding assistants (Lovable, Bolt, Cursor, Claude Code, Replit, v0, etc.) to build apps. It eliminates chaotic, fragmented workflows by providing a structured, AI-ready build pipeline that takes a user from a raw idea to a production-ready app.

Preflight is not a coding tool. It is the layer that precedes coding. It generates everything an AI coding agent needs to build well: research context, UI design prompts, PRDs, system instructions, rules files, and sequential build prompts — all scoped to the user's specific project.

**Tagline:** "Your launchpad. Every build."  
**Aesthetic codename:** The Obsidian Cockpit

---

## 2. Target Users

### Primary: The Solo Vibe Coder (53% of community)
- Mixed technical proficiency (novice to intermediate)
- Builds solo using Lovable, Bolt, or Cursor
- Pain: context loss, brittle prompts, messy desktop, no structure before coding
- Goal: ship MVPs fast without breaking things

### Secondary: The Technical Builder (31% of community)
- High technical proficiency
- Uses Cursor, Claude Code, v0
- Pain: context rot in large projects, no structured rules files, documentation gaps
- Goal: scalable, maintainable codebases with AI assistance

### Tertiary: The Non-Technical Founder (22% of community)
- Low technical proficiency
- Uses Lovable, Bolt, Replit
- Pain: doesn't know what to tell the AI, no workflow structure
- Goal: build a product without a developer

## 3. Core Features

### 3.1 Project Management
- Create unlimited projects (local-first, stored in IndexedDB via Dexie.js)
- Each project has: name, description, status, target platforms, tech stack, timestamps
- Project statuses: Ideation → Researching → Designing → Building → Shipped
- Project Hub: grid of cards with progress indicators, platform badges, last modified
- Splash screen on first load with HUD metadata aesthetic
- Command palette (⌘K) for global navigation

### 3.2 Brief Module
Structured project idea capture:
- App name, problem statement, target user, core features (ordered list), tech stack tags, target platform selection, inspiration/reference inputs, freeform notes
- Autosave on every keystroke
- Completion gating: CTA to generate Research Prompt only activates when name + problem + features are filled

### 3.3 Research Module
- Context node selector: choose which project data to include (Brief, Tech Stack)
- One-click generation of a deep research prompt using the active AI provider
- Platform launcher buttons: Perplexity, Gemini Deep Research, ChatGPT (opens in new tab)
- Upload research results (PDF, MD, TXT) to the project Vault
- Char count + token estimate on generated output

### 3.4 Design Module
- Platform mode selector: Stitch, v0, Figma AI, Locofy, Universal
- Context nodes: Brief + Research results (if available)
- One-click generation of UI design prompt
- Platform launcher buttons: Open Stitch, Open v0, Open Figma (opens in new tab)
- Upload design output (PNG, PDF, ZIP, Figma export) to Vault
- Output history: grid of previously uploaded designs

### 3.5 PRD & System Module
Three sub-sections on one screen (left panel = PRD, right panel = System Instructions + Rules):

**PRD:**
- Context nodes: Brief + Research + Design
- Generated as structured markdown document (rendered, not raw)
- Actions: Copy as Markdown, Download .md, Download .pdf, Regenerate
- Version stamp (e.g., v0.4.2-alpha)

**System Instructions:**
- Platform mode selector: Universal, Lovable, Cursor, Claude Code, Bolt
- Context: PRD + Brief + Tech Stack
- Generated as pre-formatted monospace text block
- Actions: Copy, Download .md, Regenerate
- Each agent has its own customizable system instructions (editable in Settings → Agent Prompts)

**Rules File:**
- Platform selector: Universal, Cursor (.cursorrules), Claude Code (CLAUDE.md)
- Context: PRD + System Instructions + Tech Stack
- Actions: Copy, Download .cursorrules, Download CLAUDE.md, Regenerate

### 3.6 Build Module (Sequential Build Engine)
- Platform mode selector: Universal, Lovable, Cursor, Claude Code, Bolt, Replit
- Context nodes: full project (Brief + Research + Design + PRD + System Instructions)
- "Generate Full Build Workflow" — generates all stages in sequence
- Build stages (auto-generated based on PRD features):
  - Stage 01: Foundation
  - Stage 02: Database & Auth
  - Stage 03: Core Architecture
  - Stage 04–N: Feature Blocks (dynamic, one per core feature from Brief)
  - Stage N+1: Integration Layer
  - Stage N+2: Audit & Refactor
  - Stage N+3: Bug Fix Cycle
  - Stage N+4: Polish
  - Stage N+5: Final Audit
  - Stage N+6: Deployment Prep
- Each stage card: stage number, name, description, status toggle (Not started / In progress / Done), collapsible prompt block, Copy button (prominent), Regenerate button
- Global progress bar: X of Y stages completed
- Stage dots row (clickable, scroll to stage)
- Export All: downloads full workflow as single .md file

### 3.7 Vault Module
- File storage per project: PDF, MD, PNG, TXT, JSON, ZIP
- Categories: Research, Designs, Exports, Other
- "Inject Context" toggle per file — marks it as active context node for generation
- Vault integrity score + active context token count
- Drag-and-drop upload zone
- Per-file actions: Download, Toggle context, Delete
- Link Cloud Drive (future feature placeholder)

### 3.8 Settings
- **AI Providers:** Add/edit/remove API keys for Anthropic, OpenAI, Google Gemini, DeepSeek, Groq, Custom endpoint. Status indicator (connected/disconnected) per provider.
- **Intelligence Routing:** Default model selector, streaming response toggle
- **Agent Prompts:** Custom system instructions editor per generation agent (Research, Design, PRD, System Instructions, Rules File, Build stages). "Reset to default" per agent.
- **Platform Launchers:** Toggle on/off + custom URL per platform
- **Appearance:** Dark / Light / System
- **Storage:** Export all projects as JSON, View usage logs, Clear all data (danger zone)
- **About:** Version, GitHub link, license

## 4. Technical Requirements

### 4.1 Architecture
- **Frontend:** React 18 + Vite + TypeScript
- **Styling:** Tailwind CSS + shadcn/ui + Radix UI primitives
- **State:** Zustand (global app state + UI state)
- **Local DB:** Dexie.js (IndexedDB) — primary data store for open-source version
- **Cloud DB (optional):** Supabase (PostgreSQL + Auth + Realtime) — for hosted version
- **AI Integration:** BYOK abstraction layer (provider interface with adapters per vendor)
- **Command Palette:** cmdk
- **Fonts:** Space Grotesk (headlines) + Inter (body) + JetBrains Mono (code)
- **Icons:** Material Symbols Outlined

### 4.2 BYOK AI Integration
```typescript
interface AIProvider {
  complete(params: { model: string; system: string; messages: Message[]; temperature?: number }): Promise<string>
  streamComplete(params: CompleteParams): AsyncGenerator<string>
}
// Adapters: AnthropicProvider, OpenAIProvider, GoogleProvider, DeepSeekProvider, GroqProvider, CustomProvider
// Keys stored in Dexie.js (open-source) or Supabase vault (hosted version)
// Never transmitted to Preflight servers
```

### 4.3 Data Model (Dexie.js schema)
```typescript
interface Project {
  id: string // UUID
  name: string
  description: string
  status: 'ideation' | 'researching' | 'designing' | 'building' | 'shipped'
  targetPlatforms: string[]
  techStack: string[]
  createdAt: number
  updatedAt: number
}

interface Brief {
  id: string
  projectId: string
  problem: string
  targetUser: string
  coreFeatures: { id: string; text: string }[]
  inspirations: string[]
  notes: string
  updatedAt: number
}

interface GeneratedArtifact {
  id: string
  projectId: string
  type: 'research_prompt' | 'design_prompt' | 'prd' | 'system_instructions' | 'rules_file' | 'build_prompt'
  platform: string
  content: string
  contextNodes: string[]
  agentSystemPromptId: string
  version: number
  createdAt: number
}

interface VaultFile {
  id: string
  projectId: string
  name: string
  size: number
  type: string
  category: 'research' | 'design' | 'export' | 'other'
  isActiveContext: boolean
  data: ArrayBuffer
  uploadedAt: number
}

interface AgentSystemPrompt {
  id: string
  agentType: string
  content: string
  isDefault: boolean
  updatedAt: number
}

interface AIProviderConfig {
  id: string
  provider: 'anthropic' | 'openai' | 'google' | 'deepseek' | 'groq' | 'custom'
  apiKey: string // encrypted
  model: string
  isDefault: boolean
}
```

### 4.4 Key Libraries
- `dexie` — IndexedDB wrapper
- `zustand` — state management
- `cmdk` — command palette
- `react-markdown` + `remark-gfm` — markdown rendering
- `react-dropzone` — file uploads
- `@anthropic-ai/sdk`, `openai`, `@google/generative-ai` — AI provider SDKs
- `file-saver` — file downloads
- `react-hot-toast` — toast notifications
- `framer-motion` — transitions

## 5. Design System Reference

See DESIGN.md (Obsidian Cockpit). Key tokens:
- `surface`: `#131318` | `surface-container-lowest`: `#0e0e13`
- `primary`: `#c5c0ff` | `secondary`: `#6edab4` | `tertiary`: `#ffb95d`
- No explicit 1px borders for structure — use surface tonal shifts only
- Glassmorphism on floating elements only (modals, palettes, tooltips)
- HUD metadata decorations in corners (version, build hash, latency)
- Noise texture (3% opacity SVG) on all content cards
